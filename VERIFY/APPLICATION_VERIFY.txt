SAIL Application Verification

Purpose

This document defines the minimum verification requirements for the five SAIL reference applications.

Application verification focuses on:

- application version
- SAIL stack version
- local application-conformance execution
- observed and expected application case counts
- deterministic structural identity
- operational identity separation
- lifecycle guards
- idempotent transitions
- certificate consistency
- structural reset behavior
- replay behavior
- export pack states
- referenced SAIL evidence metadata
- Python structural verification

Each application must be verified independently.

- repeated requirement approval
- repeated requirement realization

`repeat accepted transition -> same certificate + unchanged transition timestamp`

PASS

==================================================
CROSS-APPLICATION STRUCTURAL RESET VERIFICATION
==================================================

Changing upstream structural or governance inputs must invalidate dependent lifecycle artifacts.

Verification must confirm:

- changed requirement structure resets dependent realization
- changed policy resets requirement approval and realization
- changed measurement resets dependent trace or realization artifacts
- changed incident structure resets escalation, recovery, verification, and realization
- changed control structure resets dependent compliance artifacts
- changed federation configuration resets dependent network artifacts

Reset Invariant

`changed upstream structure -> dependent downstream certificates reset`
