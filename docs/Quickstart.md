# 🔐 **Deterministic Identity Model**

SAIL distinguishes structural identity from operational identity.

Structural identity:

`structural ID = hash(normalized structural payload)`

Operational identity:

`operational ID = runtime record identity`

Core distinction:

`structural identity != operational record identity`

Expected behavior:

`same structural payload -> same structural identity`

`separate runtime record -> separate operational identity`

The following operational values must not alter structural certificates:

* random record IDs
* creation timestamps
* update timestamps
* export timestamps
* selected interface state
* view or download mode

`federated audit allowed iff federation certified AND replay certified`

# 🧪 **Validation Source Distinction**

Applications locally execute their own application audits.

Application audit classification:

`LOCAL_APPLICATION_CONFORMANCE_EXECUTIONS`

Application audit execution:

`LOCAL_RUNTIME`

Core structural conformance, behavioral, mutation, realization-audit, and federation-network audit values exposed by applications are referenced from the validated core release.

Referenced core validation:

`REFERENCE_RELEASE_VALIDATION`

This distinction means:

* application scenarios are executed by the application
* core structural conformance executions are not rerun by every application
* core behavioral specifications are not rerun by every application
* core mutation tests are not rerun by every application
* core realization-audit cases are not rerun by every application
* core federation-network audit checks are not rerun by every application

Validation-source invariant:

`local application execution != referenced core validation`
